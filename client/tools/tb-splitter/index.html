Note: All parsing is done client-side; packets are never sent to the QBReader server.

Credit to Ophir Lifshitz for the original TB splitter script. This tool takes a tiebreaker packet parsed using the QBReader parser (in MODAQ mode) or YAPP and splits it into individual question JSONs for use with MODAQ's "Add Question" system.

Packet structure requirement: all directives (e.g. "Do not read to Team A.") must follow the question number, i.e. as part of the tossup or the bonus lead-in.
